Проект подготовлен для хакатона **Sechenov x MIPT** и состоит из:

- **Frontend** – рабочее место врача и маркетинговый лендинг на React.
- **Backend** – FastAPI‑сервис с авторизацией, управлением кейсами и интеграцией с ML/LLM.
- **ML / RAG‑модуль** – поиск по онкологическим рекомендациям и генерация обоснований с помощью LLM.

## Описание проекта

**OncoПротокол+** помогает врачу‑онкологу:

- Проверить, насколько назначенный протокол лечения соответствует текущим клиническим рекомендациям.
- Видеть **отклонения** и **обоснованные рекомендации** по корректировке.
- Работать в едином интерфейсе: чаты, мастер‑форма кейса, отчёты, история проверок, витрина протоколов.

Ключевые идеи:

- **"Second opinion" по протоколу**: быстрая автоматическая проверка и подсветка проблемных мест.
- **Прозрачное объяснение**: ссылка на источники (гайдлайны) и краткое резюме, почему протокол ок/не ок.
- **Удобное рабочее место врача**: чаты, история кейсов, карточка пациента, зона отчёта – в одном экране.

## Запуск проекта

### Предварительные требования

- **Python**: 3.11+
- **PostgreSQL**: 14+ 
- **Node.js**: 18+ 

Docker‑оркестрация не обязательна, проект можно запускать локально по частям.

### Backend

```powershell
cd D:\Code\Onco_SechHack\backend

# зависимости
pip install -r requirements.txt

После запуска:

- Swagger UI: `http://127.0.0.1:8001/docs`
- ReDoc: `http://127.0.0.1:8001/redoc`

### Frontend 

```powershell
cd D:\Code\Onco_SechHack\frontend
npm install
npm run dev
```

По умолчанию Vite поднимет фронт на `http://localhost:5173`.

Frontend по умолчанию стучится в backend по адресу из `VITE_API_URL` (см. раздел конфигурации).

### ML / RAG‑модуль (Cechenov_ml)

Модуль `Cechenov_ml` использует **LangChain + YandexGPT** и локальный FAISS‑индекс по текстам рекомендаций.

Базовый запуск:

```powershell
cd D:\Code\Onco_SechHack\Cechenov_ml
python -m venv .venv
pip install -r requirements.txt  # если отдельно выделен
python rag_agent.py
```

Дальше модуль поднимает CLI‑интерфейс: можно задавать вопросы по локальным данным, получая ответы и список источников.

## Основной функционал

### Рабочее место врача

- Личный кабинет онколога (`/doctor/*`).
- Чаты по кейсам, диалог с ассистентом.
- Мастер‑форма заполнения кейса (ФИО, диагноз, возраст, анамнез,назначения).
- История кейсов, просмотр прошлых отчётов и результатов проверок.

### Проверка протокола

- Приём структурированного кейса и назначений от врача.
- Сопоставление с актуальными клиническими рекомендациями.
- Подсветка отклонений (дозы, схемы, порядок линий терапии).
- Формирование отчёта: краткое резюме, список несоответствий, ссылки на источники.

### Лендинг

- Лендинг с описанием продукта.
- Дополнительные страницы: сравнение, команда, дорожная карта.

## Технологический стек

### Frontend

- **React 18.3 + TypeScript 5.7 + Vite 6**
- **Tailwind CSS** – дизайн‑система, базовые UI‑компоненты и платформенные компоненты.
- **react-router-dom 7** – маршрутизация.
- **zustand 5** – стейт (auth, чаты, layout).
- **react-hook-form + zod** – формы и валидация.
- **framer-motion**, **lucide-react**, **@radix-ui/react-dialog**.

Подробнее: `frontend/Readme.md`.

### Backend

- **Python 3.11**, **FastAPI 0.128.0**, **Uvicorn 0.40.0**
- **PostgreSQL 14+**, **SQLAlchemy 2.x**, **asyncpg**
- **Alembic** (под миграции), **Redis/aioredis** (кэш/лимиты при необходимости)
- **PyJWT / python-jose** – токены и безопасность
- **httpx** – внешние HTTP‑интеграции
- **langchain / langgraph / openai** – интеграция с LLM

Подробнее: `backend/Readme.md`.

### ML / RAG

- **LangChain** – построение цепочек вопрос‑ответ.
- **FAISS + HuggingFaceEmbeddings** – векторный поиск по текстам рекомендаций.
- **YandexGPT (ChatYandexGPT)** – генерация ответов в `Cechenov_ml/rag_agent.py`.
- Папка `ml/` зарезервирована под дальнейшее развитие специализированных моделей.

## ML / RAG модуль

Модуль `Cechenov_ml` реализует простой **RAG‑подход**:

- Загружает локальные `.txt`‑документы из папки `data/` (клинические рекомендации, методички).
- Разбивает их на чанки и индексирует в **FAISS**.
- Использует **HuggingFaceEmbeddings** (`all-MiniLM-L6-v2`).
- Под капотом **ChatYandexGPT** как LLM‑бэкенд.
- Отвечает на вопросы врача, возвращая:
  - сгенерированный ответ;
  - список документов‑источников.

Файл `Cechenov_ml/rag_agent.py` можно запускать отдельно для отладки и экспериментов.

Папка `ml/` оставлена под дальнейшее развитие: возможные модели оценки отклонений, скоры риска и т.п.

## Интеграции LLM и AI

В проекте заложена поддержка нескольких поставщиков LLM:

- **YandexGPT** – через `langchain_community.chat_models.ChatYandexGPT` в `Cechenov_ml`.
- **OpenAI / совместимые API** – через `openai` / `langchain` (зависимости уже в `backend/requirements.txt`).

Типовые сценарии использования:

- Генерация естественно‑языковых пояснений к результатам проверки кейса.
- Ответы на вопросы врача по текстам клинических рекомендаций.
- Потенциально – генерация черновиков отчётов и писем для пациента.

Конкретный провайдер и модель выбираются через переменные окружения (см. `.env` соответствующих модулей).
